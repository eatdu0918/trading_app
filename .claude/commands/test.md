# /project:test — 테스트 실행 및 리포트

$ARGUMENTS

## Test Agent: 전체 테스트 스위트 실행

다음 순서로 테스트를 실행하고 결과를 리포트하세요.

### 1. 타입 검사

```bash
pnpm typecheck
```

에러 있으면 파일별로 정리하여 보고. 자동 수정하지 말고 오케스트레이터에게 보고.

### 2. 단위 테스트 (Vitest)

```bash
pnpm test
```

실패한 테스트가 있으면:

- 파일명과 테스트 이름
- 실패 원인 (에러 메시지)
- 관련 구현 코드 경로

### 3. 빌드 검사

```bash
pnpm build
```

빌드 에러 있으면 스택트레이스 포함하여 보고.

### 4. E2E 테스트 (UI 변경이 있을 때만)

인자에 `--e2e` 또는 UI 관련 변경이 있을 때:

```bash
cd apps/web && pnpm e2e
```

### 5. 테스트 커버리지 리포트 (요청 시)

```bash
pnpm test:coverage
```

### 결과 리포트 형식

```

| Vitest | ✅ N/N 통과 / ❌ N개 실패 |
| Build | ✅ 성공 / ❌ 실패 |
| E2E | ✅ N/N 통과 / ⏭ 스킵 |

### 실패 항목
[실패 세부사항]

### 권장 다음 단계

```
